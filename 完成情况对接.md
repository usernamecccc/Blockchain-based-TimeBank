## 二、Dashboard API 文档
### 2.1 首页统计数据
#### 2.1.1 请求路径
/dashboard/stats
#### 2.1.2 请求方式
GET
#### 2.1.3 接口描述
获取首页统计数据，包括用户角色分布、活动状态分布及总参与人次。 无需角色 URL 校验 ，但需要登录 Token。
#### 2.1.4 请求头
| 参数名称 | 参数类型 | 是否必传 | 备注 |
|------|------|------|------|
| token | string | 必须 | 登录后获取的 JWT 令牌 |
#### 2.1.5 请求参数
无（Query 参数为空）
#### 2.1.6 响应数据
格式：JSON

| 参数名称 | 参数类型 | 是否必传 | 备注 |
|------|------|------|------|
| code | number | 必须 | 响应码，1 代表成功，0 代表失败 |
| msg | string | 非必须 | 提示信息 |
| data | object | 非必须 | 返回的统计数据对象 |

响应数据样例：

```
{
    "code": 1,
    "msg": "success",
    "data": {
        "userStats": {
            "elder": 5,
            "volunteer": 10,
            "admin": 2
        },
        "activityStats": {
            "statusDistribution": {
                "pending": 2,
                "active": 3,
                "rejected": 1,
                "expired": 0
            },
            "totalEngagement": 25
        }
    }
}
```


#### 2.1.7 响应字段说明
| 字段路径 | 类型 | 说明 |
|------|------|------|
| data.userStats | object | 用户统计数据 |
| data.userStats.elder | number | 老人（role=1）数量 |
| data.userStats.volunteer | number | 志愿者（role=2）数量 |
| data.userStats.admin | number | 管理员（role=3）数量 |
| data.activityStats | object | 活动统计数据 |
| data.activityStats.statusDistribution | object | 活动状态分布 |
| data.activityStats.statusDistribution.pending | number | 待审核活动数（status=1） |
| data.activityStats.statusDistribution.active | number | 进行中活动数（status=2） |
| data.activityStats.statusDistribution.rejected | number | 已拒绝活动数（status=3） |
| data.activityStats.statusDistribution.expired | number | 已过期活动数（status=4） |
| data.activityStats.totalEngagement | number | 总参与人次（所有活动参与人数之和） |
#### 2.1.8 错误码说明
| code | msg | 说明 |
|------|------|------|
| 1 | success | 请求成功 |
| 0 | NOT_LOGIN | 未登录或 token 失效 |
| 0 | ACCESS_RESTRICTED | 角色权限不足 |
