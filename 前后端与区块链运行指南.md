# 前后端与区块链运行指南（TimeBank / TimeCoin）

本文说明如何在本地一次性跑通：**本地区块链（Hardhat）→ Spring Boot 后端 → Vue PC 管理端**，以及常见问题。

---

## 一、环境准备

| 组件 | 说明 |
|------|------|
| **JDK** | 建议 **Java 11**（与 `timecoin-web` 的 Maven 配置一致） |
| **Maven** | 用于编译、启动后端模块 |
| **Node.js** | 建议 **18 或 20 LTS**（Hardhat 2 官方不支持 Node 22，若你当前为 v22，可先试试；出现异常再换 LTS） |
| **MySQL** | 默认后端连接 `localhost:3306`，数据库名 **`timecoin`**（不存在时可由 JDBC URL 自动创建，视配置而定） |

---

## 二、端口一览（避免冲突）

| 服务 | 默认端口 |
|------|-----------|
| Hardhat JSON-RPC | **8545** |
| Spring Boot（`timecoin-web`） | **8080** |
| Vue 开发服务器（`vue.config.js`） | **7000** |

若 **8080** 已被占用，请先关闭占用进程，或修改 `backend/timecoin-web/src/main/resources/application.properties` 中的 `server.port`。

---

## 三、本地区块链（Hardhat）——时间币合约

链上功能依赖 **`127.0.0.1:8545`** 上有节点；**关掉 Hardhat 进程后，本地链状态通常会清空**（内存链），重启后需重新部署或依赖启动脚本重新部署。

### 3.1 首次安装依赖

```powershell
cd D:\Blockchain-based-TimeBank\TimeCoinSystem
npm install
```

### 3.2 启动节点（终端 A，保持运行）

```powershell
cd D:\Blockchain-based-TimeBank\TimeCoinSystem
npx hardhat node
```

看到 **`Started HTTP and WebSocket JSON-RPC server at http://127.0.0.1:8545/`** 即成功。  
窗口中会列出 **Account #0～#19**：这是 Hardhat **内置测试以太坊账号**（本地假 ETH），与「业务系统里注册用户数」不是一回事。

### 3.3 部署 TimeCoin 合约（终端 B）

在 **Hardhat 已在运行** 的前提下：

```powershell
cd D:\Blockchain-based-TimeBank\TimeCoinSystem
npx hardhat deploy --network localhost
```

记下输出的 **合约地址**。若与后端配置不一致，需改后端（见下文）。

### 3.4 与后端对齐的配置

编辑：`backend/timecoin-web/src/main/resources/application.properties`

- **`blockchain.enabled=true`** — 启用 Web3j 与合约交互  
- **`blockchain.rpc-url=http://127.0.0.1:8545/`** — 与本节点一致  
- **`blockchain.contract-address=`** — 与部署输出的 TimeCoin 地址一致（仓库内默认常为 Hardhat 首次部署地址）  
- **`blockchain.private-key=`** — 须与 **部署合约所用账户** 一致；本地 Hardhat 默认常用 **Account #0** 对应私钥（仅用于开发，勿用于主网）

> **说明：** `application-dev.properties`（`spring.profiles.active=dev`）里通常为 **`blockchain.enabled=false`**。若你用 dev + H2，链上功能不会初始化；要用链时请使用默认 **`application.properties`**（MySQL），或自行在 dev 中打开链并配置 RPC。

## 四、后端（Spring Boot）

### 4.1 启动前检查

1. **MySQL** 已启动，账号密码与 `application.properties` 中 **`spring.datasource.*`** 一致。  
2. **Hardhat** 已启动（若需要链上功能）。  

### 4.2 启动方式（任选）

- **IDEA**：运行 `timecoin-web` 模块的主类 `TimecoinWebApplication`。  
- **命令行**（在仓库根目录或 `backend/timecoin-web`）：

```powershell
cd D:\Blockchain-based-TimeBank\backend\timecoin-web
mvn spring-boot:run
```

成功后浏览器访问：`http://localhost:8080`（具体以控制台为准）。

---

## 五、前端（Vue PC）

### 5.1 安装依赖（首次）

```powershell
cd D:\Blockchain-based-TimeBank\vue-system\pc
npm install
```

### 5.2 启动开发服务

```powershell
npm run serve
```

默认 **`http://localhost:7000`**。请求后端 API 的地址见 `vue-system/pc/src/utils/request.js` 中的 **`baseURL`**（通常为 **`http://localhost:8080`**），须与后端端口一致。

### 5.3 登录与菜单

使用 **管理员账号**（角色 **role = 3**）登录后可访问：**时间币管理**（链上余额 / Mint / Transfer）、**系统管理**（本机指标）等。

## 六、推荐启动顺序（便于排查）

1. **MySQL**  
2. **`npx hardhat node`**（需要时间币链上功能时）  
3. **`npx hardhat deploy --network localhost`**（新链或合约地址变更后）  
4. **Spring Boot（timecoin-web）**  
5. **`npm run serve`**（Vue）

---

## 七、链与业务数据的关系（简要）

| 位置 | 内容 |
|------|------|
| **Hardhat** | 本地区块链状态：合约余额、交易回执；**关节点易清空** |
| **MySQL** | 用户、活动等业务数据；**与链相互独立**，不因关前端而丢失 |

合约内时间币 **`userId`** 与业务约定一般为 **`用户表主键 id` 的字符串**，不是 Hardhat 列出的 `0x…` 测试地址（后者主要用于签名与 Gas）。

---

## 八、常见问题

1. **`Failed to connect to /127.0.0.1:8545`**  
   - 未启动 **`npx hardhat node`**，或端口被防火墙/代理拦截。

2. **`Port 8080 was already in use`**  
   - 关闭占用 8080 的进程，或修改后端 `server.port`。

3. **`npm install` 很慢**  
   - 可切换国内 npm 镜像后再安装依赖。

4. **Hardhat 提示 Node 版本不支持**  
   - 优先换 **Node 18/20 LTS**；若当前仍能部署与出块，可先继续用。

## 十、关闭项目后重新运行（速查）

关机、关 IDEA、关掉终端后，下次按下面顺序重新启动即可（路径按本仓库默认 **`D:\Blockchain-based-TimeBank`**）。

### 10.1 顺序总览

**MySQL → Hardhat 节点 →（按需）部署合约 → 后端 → 前端**

### 10.2 逐步指令（可复制）

**① MySQL**  
在系统中启动 MySQL 服务（与你平时用法一致）。

**② 本地区块链（需要时间币链上功能时执行）**

终端 1 — 保持窗口不关：

```powershell
cd D:\Blockchain-based-TimeBank\TimeCoinSystem
npx hardhat node
```

终端 2 — 待终端 1 已成功监听 **8545** 后再执行：

```powershell
cd D:\Blockchain-based-TimeBank\TimeCoinSystem
npx hardhat deploy --network localhost
```

若输出的 **合约地址** 与 `backend/timecoin-web/src/main/resources/application.properties` 里的 **`blockchain.contract-address`** 不一致，先改配置再启动后端。

**③ 后端**

```powershell
cd D:\Blockchain-based-TimeBank\backend\timecoin-web
mvn spring-boot:run
```

或在 IDEA 中运行 **`TimecoinWebApplication`**。

**④ 前端**

```powershell
cd D:\Blockchain-based-TimeBank\vue-system\pc
npm run serve
```

浏览器访问 **`http://localhost:7000`**（以控制台为准），使用 **管理员**账号登录。

### 10.3 不需要链上功能时

可跳过 **②**。此时请将 `application.properties` 中 **`blockchain.enabled=false`**，否则后端仍会尝试连接 **8545**；若仍要保持 **`blockchain.enabled=true`**，则必须启动 Hardhat。

### 10.4 说明

- **`TimeCoinSystem`** 若已执行过 **`npm install`**，日常重启一般**不必**重复 install，除非换了 Node 或删了 `node_modules`。  
- **`vue-system/pc`** 同理，依赖未删则可直接 **`npm run serve`**。
