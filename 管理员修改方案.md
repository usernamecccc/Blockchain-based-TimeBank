# 管理员系统优化计划

## 一、现状分析

目前管理员系统存在以下主要问题：

- **前端硬编码**：`HomeView.vue` 中的统计数字（注册人数、活动数等）、图表数据（ECharts）、通知公告、甚至管理员姓名（张三）全是写死的。
- **功能空置**：`IndexView.vue` 左侧菜单栏有很多占位按钮（如服务管理、权限管理、统计等）点击无反应。
- **后端缺失统计接口**：目前的 `ActivityController` 和 `ManageUserController` 仅支持基础的增删改查，缺乏为仪表盘（Dashboard）提供汇总统计数据的接口。

## 二、优化计划

### 1. 后端接口开发（提供数据支撑）

- **新增统计接口**：建议新建 `StatisticsController`，提供一个聚合接口（如 `/api/admin/dashboard/stats`），返回：
  - 基础统计：用户总数、志愿者人数、老人人数、活动总数、时间币流通总量。
  - 活动状态分布：各状态（待审核、报名中、进行中、已结束）的活动数量。
  - 趋势数据：近6周/月的用户增长或活动参与趋势（用于填充柱状图）。
- **扩展 Mapper 层**：在 `UserMapper` 和 `ActivityMapper` 中增加 `count` 类查询语句。

### 2. 前端界面优化（动态化与清理）

- **菜单清理**：在 `IndexView.vue` 中删除或隐藏暂未实现功能的菜单项，保持界面整洁。
- **数据对接**：在 `HomeView.vue` 的 `mounted` 生命周期中调用后端统计接口，更新 `homeData` 和图表实例。
- **快捷键功能化**：将首页的“快捷入口”卡片与现有的路由（如用户管理、活动管理）进行绑定。

## 三、双人分工方案建议

为了高效协作，建议按“前后端分离”或“模块化”进行分工。

### 同学 A：后端逻辑与数据专家

**核心任务**：

1. **数据库调研**：基于 `schema-timecoin.sql` 编写统计逻辑所需的 SQL 语句。
2. **Dashboard 接口实现**：
   - 在 `timecoin-web` 模块创建 `DashboardController`。
   - 实现 `getHomeStats` 方法，汇总各表数据。
3. **Service 层整合**：在 `ActivityService` 和 `UserService` 中封装统计逻辑。
4. **API 文档**：提供接口定义（URL、参数、返回格式），方便同学 B 对接。

### 同学 B：前端体验与视觉优化

**核心任务**：

1. **界面“大扫除”**：
   - 修改 `IndexView.vue`，移除“服务管理”、“权限管理”、“统计”、“设置”等空菜单。
2. **首页动态化**：
   - 修改 `HomeView.vue`，删除硬编码的 `homeData` 初始值。
   - 使用 `axios` 对接同学 A 提供的统计接口。
3. **图表同步**：
   - 将接口返回的数组实时填充到 ECharts 的 `series.data` 中。
4. **快捷入口修复**：
   - 为首页的“快捷入口”添加 `@click` 事件，实现点击跳转到对应的管理页面。

## 四、后续开发建议

1. **数据一致性**：统计数据建议增加缓存机制（如后期引入 Redis），或者在后端使用简单的聚合查询。
2. **Mock 数据**：在后端接口未完成前，同学 B 可以先使用 Mock 数据进行前端开发，确保样式和图表渲染正常。
3. **用户信息**：管理员首页的“张三”和“技术部”可以从现有的 `/info` 接口获取，无需重复开发获取个人信息的逻辑。
